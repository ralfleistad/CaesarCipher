The Caesar cipher is one of the simplest and well known encryption algorithms there is. The basic principle of the cipher is to shift every letter of a word a certain amount to either the right or the left. In modern practice plaintext encrypted with this cipher would not be considered secure, however at it's time is was highly effective.


overview

How does it work?

The Caesar cipher takes a message and shifts every letter with the amount of the key.
For example lets take the letter 'A' and use 6 as the key, by shifting the 'A' six places it would become 'G'. This is applied to the entire message and will give you a encrypted message that earlier was easily readable in plaintext.

How is it implemented?

To do this I utilized JavaScripts built in function to get the ASCII code for each character in the message. Thereon I had to write a custom function for the modulus operator since JavaScript does not return the correct remainder. Further I used an algorithm that uses that remainder and standard ASCII codes for upper and lowercase letters to automatically shift each of the letters.
I've implemented it so that if you shift and a letter would need to go further than 'Z', it loops back around to the begining of the alphabet and keeps shifting from there.

terminology

Caesar - When talking about Caesar it I refer to Julius Caesar, who was the first in recorded history to use this scheme.

Cipher - In cryptography this is a series of defined steps that can be followed to either encrypt or decrypt a message.

Algorithm - A process where either you or a machine conduct a series of steps to do calculations or process information.

Shift - Replacing an alphabetical character with another that corresponds to the key.

Modulus - This is an operation that findes the remainder after dividing two numbers.
